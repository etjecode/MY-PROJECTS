JavaScript Variables

Variables are containers for storing data values.

4 Ways to Declare a JavaScript Variable

var

  • The var keyword is used in all JavaScript code from 1995 to 2015.
  • Variables defined with var can be Redeclared.
  • If you re-declare a JavaScript variable declared with var, it will not lose its value.
  • Variables declared with the var keyword can NOT have block scope.
  • You can use the variable defined with var before it is declared.

When to use JavaScript var

  • The var keyword is used in all JavaScript code from 1995 to 2015.
  • The let and const keywords were added to JavaScript in 2015.
  • If you want your code to run in older browser, you must use var.

let

  • The let keyword was introduced in ES6 (2015).
  • Variables defined with let cannot be Redeclared.
  • You cannot accidentally redeclare a variable.
  • Variables defined with let have Block Scope.
  • Variables defined with let must be Declared before use.

When to use JavaScript let

  • If you want a general rule: always declare variables with const.
  • If you think the value of the variable can change, use let.

const

  • The const keyword was introduced in ES6 (2015).
  • Variables defined with const cannot be Redeclared.
  • Variables defined with const cannot be Reassigned.
  • JavaScript const variables must be assigned a value when they are declared.
  • Variables defined with const have Block Scope.

When to use JavaScript const

As a general rule, always declare a variable with const unless you know that the value will change.

    Use const when you declare:
  • A new Array
  • A new Object
  • A new Function
  • A new RegExp

Note: It's a good programming practice to declare all variables at the beginning of a script.

Block Scope

  • Before ES6 (2015), JavaScript had only Global Scope and Function Scope.
  • ES6 introduced two important new JavaScript keywords: let and const.
  • These two keywords provide Block Scope in JavaScript.
  • Meaning: Variables declared inside a { } block cannot be accessed from outside the block.
  • Variables declared with the var keyword can NOT have block scope.
  • Meaning: Variables declared inside a { } block can be accessed from outside the block.

Re-declaring JavaScript Variables

  • Variables defined with var can be Redeclared.
  • If you re-declare a JavaScript variable declared with var, it will not lose its value.
  • However, redeclaring a variable using the var keyword can impose problems.
  • Redeclaring a variable inside a block will also redeclare the variable outside the block.
  • Redeclaring a variable using the let keyword can solve this problem.
  • You cannot re-declare a variable declared with let or const.
  • Redeclaring a variable with let inside a block will not redeclare the variable outside the block.
  • Redeclaring a JavaScript variable with var is allowed anywhere in a program.
  • With let, redeclaring a variable in the same block is NOT allowed.
  • However, redeclaring a variable with let, in another block, IS allowed.

Hoisting

  • Variables defined with var are hoisted to the top and can be initialized at any time.
  • Meaning: You can use the variable before it is declared.
  • Variables defined with let and const are also hoisted to the top of the block, but not initialized.
  • Meaning: Using a let or const variable before it is declared will result in a ReferenceError.

JavaScript Identifiers

The general rules for constructing names for variables (unique identifiers) are:

  • Names can contain letters, digits, underscores, and dollar signs.
  • Names must begin with a letter (A-Z or a-z).
  • Names can also begin with a dollar sign ($) or an underscore (_).
  • Names are case sensitive.
  • Reserved words (like JavaScript keywords) cannot be used as names.

Notes:

  • Numbers are not allowed as the first character in names.
  • JavaScript treats $ and _ as letters.
  • Using the dollar sign is not very common in JavaScript, but professional programmers often use it as an alias for the main function in a JavaScript library.
  • Using the underscore is not very common in JavaScript, but a convention among professional programmers is to use it as an alias for "private (hidden)" variables.

Constant Objects and Arrays

The keyword const is a little misleading. It does not define a constant value. It defines a constant reference to a value. Because of this you can NOT:

  • Reassign a constant value
  • Reassign a constant array
  • Reassign a constant object

But you CAN:

  • Change the elements of constant array
  • Change the properties of constant object

Constant Arrays

You can change the elements of a constant array but you can NOT reassign the array.

Constant Objects

You can change the properties of a constant object but you can NOT reassign the object.
